<li>To edit the cursors you will need <code>inkscape</code> installed and to build and the render the cursor set you'll need <code>python-pil</code> and <code>x11-apps</code> installed.</li>
</ul>
<h2><a id="user-content-render-scripts" class="anchor" aria-hidden="true" href="#render-scripts"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Render Scripts</h2>
<p>For simplified development, has various scripts to render and build the cursor set are provided:</p>
<ul>
<li>
<p><a href="/madsrh/yaru/blob/master/icons/src/cursors/render-cursors.py"><strong>render-cursors.py</strong></a> will render the cursor PNG assets into <a href="/madsrh/yaru/blob/master/icons/src/cursors/bitmaps">bitmaps</a> at the appropriate sizes; run by passing the source filename to it: <code>./render-cursors.py source-cursors.svg</code>
Inside the Bitmaps folder you'll find folders containing the rendered <code>.png</code> files (24x24, 32x32,...). These must be removed, but the <code>.in</code> files are needed for the script to run.</p>
</li>
<li>
<p><a href="/madsrh/yaru/blob/master/icons/src/cursors/x11-make.sh"><strong>x11-make.sh</strong></a> builds the cursor assets into a Xcursor set and renders them to <code>/icons/Suru/cursors/</code>. These files have no extension.</p>
</li>
<li>
<p><a href="/madsrh/yaru/blob/master/icons/src/cursors/w32-make.sh"><strong>w32-make.sh</strong></a> builds the cursor assets into a Windows cursor set and renders them to <code>/icons/Suru/cursors/</code>. These files have the <code>.cur</code>extension.</p>
</li>
</ul>
<h2><a id="user-content-cursor-svg-source" class="anchor" aria-hidden="true" href="#cursor-svg-source"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Cursor SVG source</h2>
<p>The <a href="/madsrh/yaru/blob/master/icons/src/cursors/source-cursors.svg">source SVG</a> for the cursors is laid out in such a way to make editing/creating cursors simple, with a variety of layers:</p>
<ul>
<li><code>hotspots</code> shows the exact point where the cursor is active within the 24x24 region (hidden by default)</li>
<li><code>slices</code> is read by the render script and each 24x24 square had an ID that is the cursor file name (hidden by default)</li>
<li><code>cursors</code> pretty self-explanatory, these are the drawn cursors</li>
<li><code>labels</code> are just labels</li>
</ul>
<p>Both the busy cursors (the large and pointer versions) require 60 different assets to achieve a 60 FPS animation when compiled (each variant is a 6¬∞ rotation of the busy indicator).</p>
<h2><a id="user-content-css-cursors-test" class="anchor" aria-hidden="true" href="#css-cursors-test"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>CSS Cursors test</h2>
